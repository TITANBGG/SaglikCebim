## 🚨 Kritik Bulgular

### 1. Anamnez route zinciri hatalı
**Etki:** Yüksek  

- `anamnesis.py` dosyasında router kendi prefix’ini taşıyor
- `main.py` içinde aynı prefix ikinci kez ekleniyor
- Sonuç olarak gerçek route’lar çift prefix ile oluşuyor

**Fiili route örneği:**
```text
/api/v1/anamnesis/api/v1/anamnesis/profile
```

**Beklenen:**
```text
/api/v1/anamnesis/profile
```

Bu nedenle frontend’den yapılan normal anamnez istekleri backend ile eşleşmiyor.

---

### 2. Prod ortamında anamnez istekleri bir kez daha bozuluyor
**Etki:** Yüksek  

- Frontend prod’da base URL olarak `"/api"` kullanıyor
- Anamnez sayfaları ise zaten `"/api/v1/anamnesis/..."` çağrısı yapıyor
- Bu birleşim prod’da `"/api/api/v1/anamnesis/..."` üretiyor

**Sonuç:**  
Prod reverse proxy ile anamnez ekranları doğrudan hataya düşebilir.

---

### 3. Dev proxy anamnez için tanımlı değil
**Etki:** Orta-Yüksek  

`vite.config.js` içinde şu prefix’ler proxylanıyor:
- `/auth`
- `/reports`
- `/health`
- `/articles`
- `/notifications`
- `/radiology`

Ama `/api/v1/anamnesis` için proxy tanımı yok.

---

### 4. Radyoloji analiz akışı DB model uyumsuzluğu nedeniyle çöküyor
**Etki:** Yüksek  

- `RadiologyContextLink` modelinde `user_id` alanı tanımlı değil
- Buna rağmen `radiology.py` içinde nesne oluşturulurken `user_id=current_user.id` veriliyor
- Bu durum doğrudan `TypeError` üretiyor

**Sonuç:**  
Model inference başarılı olsa bile analiz sonucunun DB’ye yazıldığı aşamada akış kırılıyor.

---

### 5. Radyoloji frontend şu an bilinçli olarak kapatılmış
**Etki:** Yüksek  

- `App.jsx` içinde radiology route yorum satırına alınmış
- Sebep notu olarak “JSX build issue resolved edilene kadar disabled” bırakılmış
- Bu nedenle kullanıcı arayüzünde modül görünse bile route seviyesi kapalı

---

### 6. Pre-radyoloji formunda hatalı import var
**Etki:** Yüksek  

- `PreRadiologyForm.jsx` dosyası `../api` içinden `apiCall` import ediyor
- Bu dosya yapıda bulunmuyor
- Aynı projede gerçek API client `src/services/api.js`

---

### 7. Anamnez guard mantığı backend response ile tam uyumlu değil
**Etki:** Orta  

- `ProtectedRoute` içinde `user.has_basic_anamnesis` kontrolü var
- Ancak `/auth/me` response modeli olan `UserOut` bu alanları dönmüyor
- Bu nedenle wizard yönlendirme mantığı beklenen şekilde işlemeyebilir

---

### 8. Anamnez profile ekranında eksik route ve tanımsız handler sorunları var
**Etki:** Orta  

- Profil ekranında bazı butonlar `"/anamnesis/edit"` yoluna gidiyor
- Ama tanımlı route `"/anamnesis/edit/:type/:id"`
- Ayrıca dosyada `handleDeleteAllergy` ve `handleDeleteFamilyHistory` kullanılıyor, ama tanımı görünmüyor

---

### 9. Local çalışma port bilgileri tutarsız
**Etki:** Orta  

- `setup.ps1` parametresi `BackendPort = 8000`
- Aynı script çıktı mesajında backend’i `8001` olarak yazıyor
- `vite.config.js` ise backend proxy için `8000` kullanıyor

Bu durum ekip içinde “hangi port doğru?” karışıklığı yaratır.
